🔒 Confidentialité & RGPD

Politique de confidentialité

MathsXP · Éditeur de CCF — Version 2.0 — 2026 — Tom Rougeaud

✅ En résumé : Toutes vos données restent sur votre appareil. Aucun serveur ne les reçoit. Aucun cookie. Aucun tracker. Aucune publicité.

1. Que fait cette application avec vos données ?

MathsXP · Éditeur de CCF est un fichier HTML autonome. Toutes les informations que vous saisissez (noms d'élèves, notes, grilles, convocations, logos, signature) sont stockées uniquement dans la mémoire locale de votre navigateur, sur votre propre appareil.

Il n'existe aucun serveur propre à cette application. Vos données ne transitent jamais sur Internet.

3. Correction vocale — ce qui transite vraiment

⚠️ Point d'attention : la reconnaissance vocale est traitée par votre navigateur (Google pour Chrome, Microsoft pour Edge). L'audio de vos commandes vocales (chiffres, mots-clés) transite vers leur service. Aucune donnée élève ne transite.

Concrètement :

  • ✅ Vos chiffres énoncés (0, 1, 2, 3) et vos commandes ("Début Correction") → transités pour traduction
  • ❌ Noms d'élèves → jamais transmis
  • ❌ Notes, grilles, corrections → jamais transmis
  • ❌ Enregistrement audio → jamais effectué par cette application

La correction vocale est optionnelle. La correction manuelle ne fait transiter aucune donnée.

4. Ressource externe — Google Fonts

La police de caractères (Plus Jakarta Sans) est chargée depuis fonts.googleapis.com lors du premier accès avec connexion. Votre adresse IP est transmise à Google (requête HTTP standard). Aucune donnée applicative n'est concernée. Après le premier chargement, le navigateur utilise le cache.

5. Hébergement (GitHub Pages / Forge des Communs)

Le serveur d'hébergement voit uniquement : adresse IP du visiteur, date d'accès, navigateur. Il ne reçoit jamais les données de l'application (élèves, notes, grilles).

6. Cookies

✅ Aucun cookie. L'application utilise localStorage, qui est différent des cookies et n'est pas transmis aux serveurs. Aucun bandeau cookie n'est requis.

7. Base légale RGPD

Traitement fondé sur l'article 6.1.e du RGPD — mission d'intérêt public (évaluation scolaire obligatoire, Éducation nationale française). L'enseignant est responsable de traitement. Le développeur n'accède jamais aux données.

8. Vos droits

Accès, rectification, effacement, limitation, opposition : contacter l'enseignant évaluateur ou le DPO de l'établissement scolaire. Toutes les données peuvent être supprimées via l'onglet Sauvegarde → Réinitialiser.

💾 Données & Sauvegarde

Comprendre le stockage de vos données

Guide simple à destination de tous les utilisateurs

🗂️ Où sont stockées vos données ?

✅ Dans votre navigateur, sur votre appareil — nulle part ailleurs.
L'application n'a pas de serveur. Vos données ne quittent jamais votre ordinateur.

Techniquement, l'application utilise le localStorage de votre navigateur. C'est une zone de mémoire locale, un peu comme un petit tiroir dans votre navigateur, accessible uniquement depuis votre appareil.

📦 Qu'est-ce qui est stocké ?

  • Vos classes et élèves : noms, prénoms, tiers-temps
  • Vos grilles CCF : questions, barèmes, chapitres du BO
  • Vos corrections : niveaux 0-3 par question par élève
  • Vos convocations : groupes, dates, salles
  • Votre configuration : logos, signature, nom évaluateur
  • Vos sujets PDF : fichiers convertis en texte encodé

Tout cela est regroupé sous une seule clé technique : eccf_v5.

⚠️ Risque important : si vous videz les données de votre navigateur, désinstallez Chrome/Edge, ou changez d'ordinateur, toutes vos données sont perdues. Exportez régulièrement via l'onglet Sauvegarde !

💾 Comment fonctionne l'export (sauvegarde JSON) ?

En cliquant sur "Exporter les données" dans l'onglet Sauvegarde, l'application télécharge un fichier .json sur votre ordinateur. Ce fichier contient toutes vos données — c'est votre sauvegarde complète.

Format du nom du fichier

CCF_backup_NomEvaluateur_2025_2026-04-09.json

Il est nommé automatiquement avec votre nom et la date du jour.

Bonnes pratiques pour la sauvegarde

  • ✅ Exporter au moins une fois par semaine pendant la période des CCF
  • ✅ Garder le fichier dans un dossier sécurisé (votre espace académique, clé USB chiffrée)
  • ✅ Nommer le fichier avec la date pour vous repérer
  • ❌ Ne pas déposer le fichier dans un dossier Google Drive ou OneDrive partagé publiquement
  • ❌ Ne pas envoyer le fichier par e-mail sans protection (il contient des données élèves)

📥 Comment fonctionne l'import (restauration) ?

En cliquant sur "Importer des données", vous sélectionnez un fichier JSON précédemment exporté. Attention : l'import remplace toutes les données actuelles. Commencez toujours par exporter d'abord si vous voulez conserver vos données actuelles.

🔄 Les données se synchronisent-elles entre appareils ?

❌ Non. Il n'y a aucune synchronisation automatique entre appareils. Pour travailler sur un autre ordinateur, vous devez : exporter le JSON sur l'appareil A, puis l'importer sur l'appareil B.

🗑️ Comment supprimer toutes les données ?

Dans l'onglet Sauvegarde : bouton "Réinitialiser toutes les données". Une double confirmation est demandée. Cette action est irréversible — exportez d'abord.

📊 Combien de données peut-on stocker ?

Le localStorage est limité à environ 5 Mo par navigateur. Pour une utilisation normale (quelques classes, quelques grilles, des logos en basse résolution), cette limite n'est pas atteinte. Les sujets PDF joints (base64) peuvent occuper de l'espace. En cas de dépassement, un message d'avertissement s'affiche.

🔒 Ces données sont-elles protégées ?

Les données du localStorage ne sont accessibles que depuis votre navigateur, sur votre appareil. Personne d'autre (pas même le développeur) ne peut y accéder à distance. En revanche, si quelqu'un a accès physique à votre ordinateur et à votre session navigateur, il pourrait potentiellement y accéder. Traitez votre ordinateur comme un support confidentiel.

Base légale RGPD : traitement fondé sur l'article 6.1.e (mission d'intérêt public). L'enseignant est responsable de traitement pour les données saisies dans l'outil. Pour exercer vos droits (accès, rectification, effacement), contacter directement l'enseignant.
