🔒 À propos des données

À propos des données

MathsXP · Éditeur de CCF — Version 2.0 — 2026 — Tom Rougeaud

✅ En résumé : Toutes vos données restent sur votre appareil. Aucun serveur ne les reçoit. Aucun cookie. Aucun tracker. Aucune publicité.

1. Où sont stockées vos données ?

MathsXP · Éditeur de CCF est un fichier HTML autonome. Toutes les informations saisies (noms d'élèves, notes, grilles, convocations, logos, signature) sont stockées uniquement dans la mémoire locale de votre navigateur, sur votre propre appareil, via l'API localStorage.

Il n'existe aucun serveur propre à cette application. Vos données ne transitent jamais sur Internet.

⚠️ Risque de perte : si vous videz les données de votre navigateur, changez de profil ou d'appareil, toutes les données sont perdues définitivement. Exportez régulièrement via l'onglet Sauvegarde.

2. Qu'est-ce que le localStorage ?

Le localStorage est une zone de mémoire propre à votre navigateur, sur votre ordinateur. C'est comme un petit tiroir numérique accessible uniquement depuis votre appareil. Il est totalement invisible depuis l'extérieur : ni le serveur qui héberge le programme, ni Google, ni personne d'autre ne peut y accéder.

Toutes vos données (classes, élèves, grilles, corrections) sont regroupées sous la clé technique eccf_v5. Voici à quoi ressemble un extrait de données sauvegardées :

{
  "params": { "evaluateur": "M. Rougeaud", "session": "2026" },
  "classes": {
    "TMCVA 2026": { "eleves": [{ "nom": "DUPONT Marie", "tiersTemps": false }] }
  },
  "evals": { "grille-001": { "eleve-abc": { "apps": { "q1": 2, "q2": 3 } } } }
}

4. Correction vocale — ce qui transite vraiment

La correction vocale utilise la Web Speech API du navigateur. Voici exactement ce qui se passe, sans ambiguïté :

DonnéeStockée durablement dans le navigateur ?Transmise sur Internet ?Qui peut y accéder ?
Flux audio brut (ce que vous dites)❌ Non — jamais⚠️ Oui, vers Google/MicrosoftGoogle (Chrome) ou Microsoft (Edge) — pour la reconnaissance uniquement
Transcription textuelle (ex : "deux", "valider")❌ Non — traitée en mémoire vive uniquement❌ NonL'application uniquement, sur votre appareil
Noms d'élèves, notes, grilles✅ Oui, dans localStorage❌ JamaisVous seul, sur votre appareil
Concrètement : quand vous dites "deux" ou "valider correction", cet audio est envoyé au service de Google ou Microsoft qui le traduit en texte et vous renvoie "deux". Aucun nom d'élève, aucune note, aucune donnée scolaire ne transite. L'audio lui-même n'est ni stocké par l'application, ni stocké durablement par Google/Microsoft (ils le traitent en temps réel et ne l'utilisent pas pour le ciblage publicitaire).

La correction vocale est entièrement optionnelle. En correction manuelle, rien ne transite.

Lorsque vous fermez le mode vocal, le microphone est libéré immédiatement (l'icône rouge disparaît de l'onglet du navigateur).

5. Export/Import JSON — sauvegarde de vos données

L'export télécharge un fichier .json sur votre ordinateur contenant l'intégralité de vos données. Ce fichier contient des données personnelles. Il n'est envoyé nulle part automatiquement.

✅ Bonnes pratiques❌ À éviter
Stocker dans un espace à accès restreintDéposer sur un cloud public (Drive public, etc.)
Exporter au moins une fois par semaineEnvoyer par e-mail non chiffré
Supprimer les sauvegardes obsolètes en fin d'annéeGarder des archives de plusieurs années sans nécessité

6. Supprimer vos données de manière permanente

Le bouton "Réinitialiser" de l'onglet Sauvegarde efface le localStorage — mais Ctrl+Z peut rétablir les données si vous ne fermez pas le navigateur. Pour une suppression vraiment permanente, suivez cette procédure :

Suppression définitive (Chrome/Edge) :
Menu ⋮ → Paramètres → Confidentialité et sécurité → Effacer les données de navigation → cocher "Cookies et données de sites" → choisir la période souhaitée → Effacer les données.

⚠️ Cela supprimera aussi les données d'autres sites. Exportez d'abord si vous souhaitez conserver vos corrections.

7. Cookie, traceurs, publicité

✅ Aucun cookie. Aucun traceur. Aucune publicité. Le localStorage n'est pas un cookie et n'est pas transmis aux serveurs.

8. Ressource externe — Google Fonts

La police de caractères est chargée depuis fonts.googleapis.com au premier chargement. Seule votre adresse IP est transmise à Google (requête HTTP standard). Aucune donnée applicative n'est concernée.

9. Hébergement (GitHub Pages / Forge des Communs)

Le serveur d'hébergement voit : adresse IP, date d'accès, navigateur. Il ne reçoit jamais les données de l'application (élèves, notes, grilles).

10. Base légale et vos droits

Traitement fondé sur l'article 6.1.e du RGPD — mission d'intérêt public (évaluation scolaire, Éducation nationale française). L'enseignant qui utilise l'outil est responsable de traitement.

Vos droits (accès, rectification, effacement, limitation, opposition) : les données étant stockées uniquement sur votre propre appareil, ces droits s'exercent directement par vous. Toutes les données peuvent être supprimées via l'onglet Sauvegarde → Réinitialiser, ou en vidant le cache de votre navigateur (voir section 6 ci-dessus).
